   - Add `("<report> — TSMIS vs TSN", _cmp_<report>, "files", "tsn")` to `COMPARE_REPORTS`.

   - Add Intersection's consolidator to `_CONSOLIDATOR_BY_SUBDIR`.
   - (Optional, if wanted) build Intersection + HL-PDF **cross-env** adapters so they

6. **Reflect everywhere** — the Everything matrix's vs-TSN mode is data-driven, so
   newly-supported reports light up automatically; confirm the Consolidate tab and

## Auditing — the new formulas must be FLAWLESS (top priority)
Every new consolidator and comparator is a discrepancy tool people will trust to make
real decisions, so correctness is non-negotiable — and we now have **both the TSN and
the TSMIS versions of every report**, which is everything needed to verify against ground
truth. For each report, before it ships:
- **Reconcile both source files by hand first** — open the TSN file and the TMSIS export,
  agree on the key columns, the roadbed/ditto rules, and what "the same row" means, BEFORE
  writing the comparator. The schema must come from the data, not a guess.
- **Prove the consolidator** preserves every route/row from the per-route files (no dropped
  or merged rows; counts match the raw inputs).
- **Prove the comparator cell-for-cell** against the real TSN/TMSIS pair: a `check_compare_<report>.py`
  golden over planted + real data, plus a **COM recalc** of the live-formulas workbook so the
  on-sheet formulas equal the values copy. Spot-check a sample of diffs by hand.
- **Self-check rows must hold** (the Summary's totals reconcile) and **no silent coercion** —
  a value that can't be parsed surfaces as a diff, never as a false match.
- Treat the first report (Ramp Detail or Highway Sequence) as the **reference recipe**: get it
  audited flawless, then reuse the exact verification harness for the rest.
