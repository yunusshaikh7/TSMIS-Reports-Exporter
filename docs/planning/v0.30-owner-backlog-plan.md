## 2. The two marathon runs

Each marathon is one continuous push (the M-B pattern): staged commits, the FULL gate
(`build/run_checks.py -j 4 -k` + byte-compile + ruff + `build.ps1 -SelfTest`) at every
stage boundary, GUI work verified in the `#mock` preview, and a tagged release at the
end. M1 is the user-facing comparison/GUI/diagnosis run; M2 is the structural run
(engine, evidence policy, new matrix, readiness). M1 ships first so the Edge diagnosis
kit reaches the work PC a full release early.
